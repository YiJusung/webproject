# 참여도(Engagement) 지표 설명

## 용어 정의

### 조회수 (Views)
- **의미**: 콘텐츠를 **본 횟수**
- **특징**: 수동적 지표 (사용자가 단순히 클릭/열람)
- **예시**: 
  - YouTube: 동영상 조회수
  - 뉴스: 기사 조회수
  - GitHub: 저장소 조회수 (제한적)

### 참여도 (Engagement)
- **의미**: 사용자가 콘텐츠에 **실제로 상호작용한 지표**
- **특징**: 능동적 지표 (사용자가 행동을 취함)
- **예시**:
  - 좋아요 (Likes)
  - 댓글 (Comments)
  - 업보트 (Upvotes)
  - 스타 (Stars)
  - 포크 (Forks)
  - 공유 (Shares)

## 현재 시스템에서 수집하는 지표

### 1. Reddit
- ✅ **참여도 지표**: 
  - `upvotes`: 업보트 수 (사용자가 좋아요를 누른 횟수)
  - `comments`: 댓글 수 (사용자가 댓글을 작성한 횟수)
- ❌ **조회수**: Reddit API에서 제공하지 않음

### 2. YouTube
- ✅ **조회수**: `views` (동영상을 본 횟수)
- ✅ **참여도 지표**:
  - `likes`: 좋아요 수
  - `comments`: 댓글 수

### 3. GitHub
- ✅ **참여도 지표**:
  - `stars`: 스타 수 (저장소를 즐겨찾기한 횟수)
  - `forks`: 포크 수 (저장소를 복사한 횟수)
  - `watchers`: 워처 수 (저장소를 관찰하는 사용자 수)
- ❌ **조회수**: GitHub API에서 일반적으로 제공하지 않음 (저장소 소유자만 접근 가능)

### 4. 뉴스 (RSS)
- ❌ **조회수**: RSS 피드에 포함되지 않음
- ❌ **참여도**: RSS 피드에 포함되지 않음

## 관심도(Interest Score) 계산 방식

현재 시스템에서는 **조회수와 참여도를 합산**하여 "관심도" 점수를 계산합니다:

```python
# YouTube
interest = views + (likes × 10) + (comments × 5)
# 조회수 + 참여도(좋아요, 댓글)

# Reddit
interest = upvotes + (comments × 2)
# 참여도만 (조회수 없음)

# GitHub
interest = (stars × 10) + (forks × 5) + (watchers × 2)
# 참여도만 (조회수 없음)

# News
interest = 1
# 기본값 (조회수/참여도 정보 없음)
```

## 용어 정리

### 현재 사용 중인 용어
- **관심도 (Interest Score)**: 조회수 + 참여도를 합산한 종합 점수
- **참여도 (Engagement)**: 사용자가 실제로 상호작용한 지표 (좋아요, 댓글, 스타 등)
- **조회수 (Views)**: 콘텐츠를 본 횟수

### 더 명확한 용어 제안
만약 혼란스럽다면 다음과 같이 구분할 수 있습니다:

1. **조회수 (Views)**: 본 횟수
2. **상호작용 (Interactions)**: 좋아요, 댓글, 공유 등
3. **관심도 (Interest Score)**: 조회수 + 상호작용을 합산한 점수

또는:

1. **수동 참여 (Passive Engagement)**: 조회수
2. **능동 참여 (Active Engagement)**: 좋아요, 댓글, 스타 등
3. **종합 관심도 (Total Interest)**: 수동 + 능동 참여 합산

## 결론

**"참여도"**는 사용자가 콘텐츠에 **실제로 행동을 취한 지표**를 의미합니다:
- 좋아요를 누름
- 댓글을 작성함
- 스타를 누름
- 포크를 함
- 업보트를 함

이러한 지표들은 단순히 "본 것"보다 더 높은 관심도를 나타냅니다.
